I have recently been working on a fair bit of PowerCLI for a Cyber Defense competition called NECCDC. Every time, when I connect to vCenter, I receive this following error.<\/p>\n

This error occurs when the default VMWare certificates are not replaced with trusted\/matching ones.<\/p>\n

To resolve this, you need to run the following command.<\/p>\n

 Set-PowerCLIConfiguration -InvalidCertificateAction Ignore -Confirm:$false<\/pre>\n
This command will ignore all invalid certs from your vCenter server. It is equivalent to clicking “Install this certificate and do not display any security warning” from the old vSphere Client.<\/p>\n
<\/p>\n

With this command set, you can not connect to your\u00a0 vCenter server and in this case, listing out the currently configured port groups.<\/p>\n
